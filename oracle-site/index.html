GPS Accuracy Engine

The Same Math as Missiles.
On Your Phone.

Raw GPS is ±15m on a good day. Inside a tunnel or next to tall buildings? Forget it. Oracle runs every position through a 10-algorithm pipeline before it touches your map.

🚀
The Kalman filter was invented by Rudolf Kálmán in 1960. First used in the Apollo navigation computer to guide astronauts to the Moon — and still the core algorithm in every cruise missile, drone autopilot, and smartphone GPS chip on Earth today.
Algorithm 01

Kalman Filter — Position + Velocity

Maintains position and velocity states per axis. Each GPS fix triggers a predict → update cycle that blends the new measurement with the motion model using optimal math that minimises mean-square error.

x⁻ = F·x   P⁻ = F·P·Fᵀ + Q K = P⁻·Hᵀ / (H·P⁻·Hᵀ + R) x = x⁻ + K·(z − H·x⁻)
Algorithm 02

Adaptive Process Noise (Q-scaling)

When you brake hard or corner sharply, the innovation spike is detected and Q scales up by 20× so the filter trusts new measurements more. It decays back to baseline automatically.

Q_eff = Q_base × q_factor q_factor ∈ [0.5, 20.0] manoeuvre → q×2, steady → q×0.85
Algorithm 03

Mahalanobis Distance Gating

Before updating, each fix is tested against the NIS (Normalised Innovation Squared). If NIS > 5.99 — the 95th percentile of a chi-square distribution with 2 DOF — the fix is a statistical outlier and rejected.

NIS = y_lat²/S_lat + y_lon²/S_lon if NIS > 5.99 → reject fix (chi-square, 2 DOF, 95th percentile)
Algorithm 04

Accuracy Gating

Any fix worse than 80 metres (cell-tower fallback quality) is hard-rejected. A good smartphone GPS fix is 3–15m. Wi-Fi assisted is 20–50m. The filter never sees garbage.

if accuracy_m > 80 → discard typical smartphone GPS: 3–15 m Wi-Fi assisted: 20–50 m
Algorithm 05

Speed-Jump Outlier Rejection

If implied speed between two fixes exceeds 90 m/s (≈200 mph), the fix is physically impossible for a road vehicle and discarded. GPS multipath off buildings causes these phantom jumps constantly.

implied_speed = Δdist / Δt if implied_speed > 90 m/s → reject (≈ 200 mph road vehicle limit)
Algorithm 06

Dead Reckoning

When a fix is rejected, position is extrapolated forward from the last known speed and heading. The same technique ships used before GPS — your dot keeps moving smoothly in tunnels and urban canyons.

Δlat = v·cos(θ)·Δt / m_per_deg Δlon = v·sin(θ)·Δt / m_per_deg error grows ~linearly with time
Algorithm 07

Circular Mean Heading

You can't average compass angles with arithmetic mean — the average of 1° and 359° should be 0°, not 180°. We convert each heading to a unit vector, sum them, and take atan2 for the correct mean.

θ_mean = atan2(Σ sin(θᵢ), Σ cos(θᵢ)) R = |mean_vector| ∈ [0, 1] R≈1: tight cluster · R≈0: scattered
Algorithm 08

EMA Speed Smoothing

Raw GPS speed spikes 10–20 km/h between consecutive fixes. An Exponential Moving Average with α=0.25 smooths these out — ~7-sample window, fast reaction to real speed changes, no noise.

v_smooth = α·v_new + (1−α)·v_prev α = 0.25 → ~7-sample window ~35 ms lag at 5 Hz GPS
Algorithm 09

Stationary Lock via Accelerometer

Parked? Raw GPS wanders 5–20m due to atmospheric noise. The accelerometer is sampled at 10 Hz. If variance over the last 1.2 seconds is below threshold, the phone is still — GPS position is frozen. Zero drift.

σ² = Σ(|aᵢ| − μ)² / N if σ² < 0.04 m²/s⁴ → stationary window: 12 samples @ 10 Hz = 1.2 s
Algorithm 10

Accuracy-Adaptive Measurement Noise R

Measurement noise R is set per-fix from the fix's reported accuracy. A perfect 5m fix gets R≈2×10⁻⁹ (trusted almost completely). A mediocre 50m fix gets R≈2×10⁻⁷. Precision drives trust.

R_lat = (accuracy / m_per_deg_lat)² 5m fix → R ≈ 2×10⁻⁹ (high trust) 50m fix → R ≈ 2×10⁻⁷ (low trust)

// 2-state Kalman: x = [position, velocity]ᵀ // F = [[1, Δt], [0, 1]] H = [1, 0] predict(dt) { const pos_ = this.pos + this.vel * dt const pp_ = this.pp + dt*(this.pv*2) + dt²*this.vv + Q*dt⁴/4 const pv_ = this.pv + dt*this.vv + Q*dt³/2 const vv_ = this.vv + Q*dt² } update(z, r) { const innov = z - this.pos // innovation = measurement − prediction const S = this.pp + r // innovation variance const K = this.pp / S // Kalman gain ∈ [0,1] this.pos += K * innov // posterior position this.pp = (1-K) * this.pp // posterior variance (shrinks) }
